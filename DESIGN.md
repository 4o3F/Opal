## High-Level Architecture
```
Client / Scheduler
        |
        v
+-------------------------------+
| Core Host                     |
| - HTTP API                    |
| - Router                      |
| - Plugin Registry             |
| - Loader / Verifier           |
| - Worker Supervisor           |
| - Capability Broker           |
| - Cache / Storage             |
| - Feed Renderer               |
| - Metrics / Audit             |
+-------------------------------+
        |
        | Typed RPC over MessagePort
        v
+-------------------------------+
| Plugin Worker                 |
| - Route handlers              |
| - Parsing / extraction        |
| - Normalization to Feed model |
+-------------------------------+
```

## Host Components
### HTTP API
- Accept requests such as `/feed/:plugin/:route`.
- Parse path parameters, query parameters, and requested output format.
- Apply authentication or rate limits for public or private deployments.
- Return RSS, Atom, or JSON Feed generated by the renderer.

### Router
- Resolve incoming requests to a plugin and route definition.
- Validate route existence and route-level input schema.
- Select the active plugin version.

### Plugin Registry
- Store plugin metadata, version history, ownership, compatibility, and status.
- Track requested capabilities and network allowlists.
- Support states such as `active`, `degraded`, `deprecated`, and `disabled`.

### Plugin Loader and Verifier
- Load only locally installed, version-pinned plugin bundles.
- Verify manifest validity, signature or checksum, and compatibility.
- Refuse runtime remote loading.
- Prepare worker startup configuration and pass a dedicated `MessagePort`.

### Worker Supervisor
- Create, pool, recycle, and terminate plugin workers.
- Apply worker-level permission policy.
- Enforce timeouts, memory limits, concurrency limits, and restart strategy.
- Isolate plugin crashes from the host.

### Capability Broker
- Expose a limited host API to plugins over typed RPC.
- Validate each capability call against plugin policy.
- Attach rate limiting, caching, credential injection, and auditing.

### Cache and Storage
- Cache upstream HTTP responses.
- Cache normalized feed results.
- Support TTLs, cache invalidation, stale-while-revalidate, and prewarming.
- Optionally persist snapshots for debugging and reproducibility.

### Feed Renderer
- Convert a normalized `FeedDocument` into RSS 2.0, Atom, or JSON Feed.
- Keep rendering behavior consistent across all plugins.
- Prevent plugins from generating raw XML directly.

### Scheduler
- Periodically refresh popular or critical feeds.
- Retry failed jobs with backoff.
- Support prewarming and background refresh.
- Measure job success, latency, and upstream instability.

### Observability and Audit
- Emit structured logs and metrics per plugin.
- Trace host capability usage.
- Record which domains a plugin accessed through host mediation.
- Expose dashboards and alerting for plugin health.
